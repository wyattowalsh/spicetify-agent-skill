# Audit an Extension

Use this workflow before enabling any extension from a local folder, GitHub
checkout, Marketplace entry, or generated scaffold.

```text
/spicetify audit this extension before enabling it
```

The audit report should identify source files, risky APIs, network behavior,
obfuscation, prompt-injection attempts in documentation, required provenance,
and whether the asset is eligible for a dry-run install plan.

Passing audit does not automatically enable the extension. It only allows the
next dry-run plan to reference the audited artifact and request confirmation.
