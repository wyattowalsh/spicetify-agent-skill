# Mode: custom-app

**Path:** `apps/docs/content/docs/archive/add-spicetify-skill/modes/custom-app.mdx`
**Purpose:** Detailed contract for `/spicetify custom-app` mode.
**Status:** Proposed
**Load/use when:** Implementing, testing, or reviewing `custom-app` behavior.

## Purpose

Scaffold, build, install, enable, debug, disable, or remove custom apps.

## Inputs

app name, manifest, React/TS scaffold settings, source path, trust level.

## Preconditions

Safe folder name; manifest and `index.js` render behavior valid.

## Commands/files touched

`CustomApps/<name>/index.js`, `manifest.json`, style/subfiles; `spicetify config custom_apps <name>|<name>-`; apply.

## Safety checks

Manifest validation; audit `subfiles` and `subfiles_extension`; snapshot before enable.

## Plan output

`CustomAppPlan` with files, manifest validation, audit/provenance, config diff, verification.

## Execution flow

Scaffold/stage → validate manifest/render → build/copy → enable → apply.

## Verification flow

Folder, manifest, `index.js`, config entry, optional navigation/log check.

## Rollback flow

Remove from `custom_apps`, restore/delete app files from snapshot.

## Idempotency notes

The planner MUST diff current state before emitting mutations. Re-running an already satisfied desired state SHOULD become a no-op plus verification unless the user explicitly requests refresh/reapply.

## Failure modes and recovery

- Manifest invalid -> do not enable.
- subfiles_extension risky -> audit as extension before apply.


## Data contracts

Primary contracts: `asset-manifest, audit-report, consent-grant`. Any implementation-specific schema expansion MUST update `apps/docs/content/docs/archive/add-spicetify-skill/schemas/index.mdx`, `api-contracts.md`, and regression prompts.

## Cross-cutting controls

- Policy decision is required before execution, even for no-op verification plans.
- Secrets, prefs content, logs, screenshots, and private paths follow `privacy-redaction.md`.
- Third-party, Marketplace, imported, or unknown assets require provenance and audit before enablement.
- Desired-state or automation inputs cannot waive non-negotiable safety invariants.

- `/spicetify scaffold a stats custom app`
