# Mode: restore

**Path:** `apps/docs/content/docs/archive/add-spicetify-skill/modes/restore.mdx`
**Purpose:** Detailed contract for `/spicetify restore` mode.
**Status:** Proposed
**Load/use when:** Implementing, testing, or reviewing `restore` behavior.

## Purpose

Restore a named snapshot with pre-restore guard snapshot.

## Inputs

`snapshotId`, `restoreScope`, `applyAfterRestore?`, `deleteUnknown?`.

## Preconditions

Target snapshot exists and validates; current state can be snapshotted.

## Commands/files touched

Config/assets files under allowed roots; optional `spicetify apply` after restore.

## Safety checks

Requires confirmation for overwrites/deletes; pre-restore snapshot required.

## Plan output

`RestorePlan` with file diff, overwrite/delete list, pre-restore snapshot, and verification.

## Execution flow

Validate snapshot; create guard snapshot; restore staged files atomically; optionally apply.

## Verification flow

Compare restored hashes/config; run doctor subset; optionally verify apply output.

## Rollback flow

Restore from guard snapshot.

## Idempotency notes

The planner MUST diff current state before emitting mutations. Re-running an already satisfied desired state SHOULD become a no-op plus verification unless the user explicitly requests refresh/reapply.

## Failure modes and recovery

- Target snapshot invalid -> stop.
- Overwrite drift -> create pre-restore snapshot and require confirmation.


## Data contracts

Primary contracts: `operation-plan, snapshot-manifest, operation-run`. Any implementation-specific schema expansion MUST update `apps/docs/content/docs/archive/add-spicetify-skill/schemas/index.mdx`, `api-contracts.md`, and regression prompts.

## Cross-cutting controls

- Policy decision is required before execution, even for no-op verification plans.
- Secrets, prefs content, logs, screenshots, and private paths follow `privacy-redaction.md`.
- Third-party, Marketplace, imported, or unknown assets require provenance and audit before enablement.
- Desired-state or automation inputs cannot waive non-negotiable safety invariants.

## Example user prompts
