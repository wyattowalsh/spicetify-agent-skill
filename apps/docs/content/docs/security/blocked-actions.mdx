# Blocked Actions

`/spicetify` refuses requests that would weaken user safety, expose private data,
or bypass platform boundaries.

## Always blocked

- Arbitrary shell execution or command passthrough.
- Secret exfiltration, token collection, or credential display.
- DRM, subscription, account, or ad-bypass behavior.
- Running install scripts from third-party READMEs or package metadata.
- Disabling snapshot, audit, provenance, redaction, or confirmation invariants.

## Manual-only or separately approved

- Package-manager commands.
- Permission changes.
- Spotify launch-flag edits.
- Publishing, deployment, analytics, hosted search, or cloud configuration.
- Copying shadcn/ui or other registry components from external sources.

When a request lands here, the safe response is a refusal of the unsafe part plus
read-only diagnostics or a human-reviewed checklist.
