## Completion rule

A mode is complete only when its happy path, blocked path, verification path, and rollback/report path are implemented against fake environments. Real Spotify/Spicetify mutation is not required for CI acceptance and must not happen in CI.

## Review loop

When a requirement changes, update in this order:

1. OpenSpec delta spec;
2. schema or mode contract;
3. task graph;
4. fixtures/tests;
5. acceptance matrix;
6. skill references and handoff docs.
