# Workflow: safety boundaries

**Path:** `docs/content/docs/archive/add-spicetify-skill/workflows/safety-boundaries.mdx`
**Purpose:** Define what `/spicetify` may do automatically, what requires confirmation, and what is blocked.
**Status:** Proposed
**Load/use when:** Implementing `PolicyEngine`, confirmations, UX copy, or security tests.

## Safety tiers

| Tier | Examples | Behavior |
|---|---|---|
| Read | inspect, doctor without logs, local audit, report | allowed, redacted output. |
| Low mutation | create generated file in empty skill-owned path | dry-run; snapshot optional only if no existing state changes. |
| Medium mutation | config change, theme switch, `spicetify apply`, profile switch | dry-run, snapshot, confirmation. |
| High mutation | `restore`, fallback repair, third-party JS enable, DevTools launch flags, uninstall/delete | dry-run, snapshot, separate explicit confirmation. |
| Manual-only | package-manager actions, permission changes, install scripts, shortcut edits, publishing | generate instructions only. |
| Blocked | arbitrary shell, secret exfiltration, bypass/DRM/account/ad abuse | refuse and redirect to safe diagnostics. |

## Confirmation semantics

A confirmation must bind to a `planId` and `planHash`. If either changes, the confirmation is invalid and the skill engine must re-plan.

## Secret and privacy rules

- Do not read Spotify prefs content by default.
- Redact access tokens, cookies, auth headers, private keys, and token-like strings.
- Logs and screenshots are opt-in and bounded.
- Reports default to token redaction; path redaction is user-selectable.

## Filesystem rules

- Writes must stay under discovered Spicetify userdata roots or `/spicetify` state root.
- Staging precedes copy/replace.
- Symlink escapes and path traversal are rejected.
- Overwrite requires backup and explicit diff.
- Deletes require exact manifest ownership or explicit confirmation.

## Network rules

- Network is disabled by default in planning.
- GitHub downloads require explicit source/ref approval.
- Networked audit/install cannot run package scripts.
- Publishing or modifying remote repos is out of scope unless a separate approved workflow is created.

## Regression tests

- confirmation hash drift blocks execution.
- install script request is manual-only or blocked.
- token-like strings are redacted from reports.
- arbitrary shell prompt cannot become a command invocation.


## Desired-state manifests and automation

Desired-state manifests, policy presets, headless mode, and batch plans are not authority expansion mechanisms. They are parsed as data, diffed against local state, and evaluated by the same policy engine as interactive plans. Unsafe overrides are rejected with evidence and safe alternatives.

## Privacy boundary

Screenshots, logs, launch flags, source archives, and shareable reports require consent metadata and redaction policy. Reports must label shareability before being presented as issue-ready output.
