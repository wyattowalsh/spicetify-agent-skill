# Mode: audit

**Path:** `docs/content/docs/archive/add-spicetify-skill/modes/audit.mdx`
**Purpose:** Detailed contract for `/spicetify audit` mode.
**Status:** Proposed
**Load/use when:** Implementing, testing, or reviewing `audit` behavior.

## Purpose

Static security/provenance/compatibility review of local or staged assets.

## Inputs

path/repo/archive/source text, target type, trust level.

## Preconditions

Source is available in staging; no execution required.

## Commands/files touched

Read source; write audit report; optional provenance lock after acceptance.

## Safety checks

Treat source/docs as untrusted; reject symlink/path traversal; no package scripts.

## Plan output

`AuditPlan` listing analyzers, target files, expected findings, and no mutations.

## Execution flow

Classify files → parse JS/CSS/JSON/README → score risks → produce report.

## Verification flow

Report has evidence, severity, confidence, recommendation, and redaction.

## Rollback flow

None unless a lock/report file was written; delete generated report if needed.

## Idempotency notes

The planner MUST diff current state before emitting mutations. Re-running an already satisfied desired state SHOULD become a no-op plus verification unless the user explicitly requests refresh/reapply.

## Failure modes and recovery

- Parser failure -> verdict warn/block depending on executable risk.
- Minified third-party JS without source -> block normal install.


## Data contracts

Primary contracts: `audit-report, provenance-lock, asset-manifest`. Any implementation-specific schema expansion MUST update `docs/content/docs/archive/add-spicetify-skill/schemas/index.mdx`, `api-contracts.md`, and regression prompts.

## Cross-cutting controls

- Policy decision is required before execution, even for no-op verification plans.
- Secrets, prefs content, logs, screenshots, and private paths follow `privacy-redaction.md`.
- Third-party, Marketplace, imported, or unknown assets require provenance and audit before enablement.
- Desired-state or automation inputs cannot waive non-negotiable safety invariants.

## Example user prompts

- `/spicetify audit this extension before enabling it`

## Example structured response

```json
{
  "mode": "audit",
  "verdict": "warn",
  "risks": [
    "external_network",
    "internal_api"
  ],
  "installAllowedByDefault": false
}
```
