# Mode: snapshot

**Path:** `docs/content/docs/archive/add-spicetify-skill/modes/snapshot.mdx`
**Purpose:** Detailed contract for `/spicetify snapshot` mode.
**Status:** Proposed
**Load/use when:** Implementing, testing, or reviewing `snapshot` behavior.

## Purpose

Capture restorable user Spicetify config/assets state before mutation.

## Inputs

`label`, `scope`, `includeThirdPartyAssets?`, `includeLogs?`, `redactionLevel`.

## Preconditions

Config/userdata roots are known; snapshot store is writable.

## Commands/files touched

Reads config, Themes, Extensions, CustomApps, /spicetify profiles/locks; writes immutable snapshot manifest/files.

## Safety checks

Skip prefs content, caches, unapproved logs, symlink escapes, and files outside allowed roots.

## Plan output

`SnapshotPlan` listing included/excluded files, estimated size, hashes, and sensitive exclusions.

## Execution flow

Resolve roots; stage file list; hash; copy to snapshot dir; write manifest; verify hashes.

## Verification flow

Re-hash snapshot; ensure manifest is complete and immutable.

## Rollback flow

Snapshot creation failure stops dependent operation; no source mutation to roll back.

## Idempotency notes

The planner MUST diff current state before emitting mutations. Re-running an already satisfied desired state SHOULD become a no-op plus verification unless the user explicitly requests refresh/reapply.

## Failure modes and recovery

- Snapshot hash mismatch -> stop before mutation.
- Sensitive file detected -> exclude and record redacted metadata.


## Data contracts

Primary contracts: `snapshot-manifest, operation-report`. Any implementation-specific schema expansion MUST update `docs/content/docs/archive/add-spicetify-skill/schemas/index.mdx`, `api-contracts.md`, and regression prompts.

## Cross-cutting controls

- Policy decision is required before execution, even for no-op verification plans.
- Secrets, prefs content, logs, screenshots, and private paths follow `privacy-redaction.md`.
- Third-party, Marketplace, imported, or unknown assets require provenance and audit before enablement.
- Desired-state or automation inputs cannot waive non-negotiable safety invariants.
