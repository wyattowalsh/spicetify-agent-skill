# Mode: watch

**Path:** `docs/content/docs/archive/add-spicetify-skill/modes/watch.mdx`
**Purpose:** Detailed contract for `/spicetify watch` mode.
**Status:** Proposed
**Load/use when:** Implementing, testing, or reviewing `watch` behavior.

## Purpose

Run bounded development watch loops for trusted local projects/themes.

## Inputs

target type/name, project path, duration, build strategy.

## Preconditions

Target is local trusted/generated; no third-party scripts without approval.

## Commands/files touched

`spicetify watch` for themes; generated-template build/watch commands only if approved.

## Safety checks

Bound duration/output; stop path; no network by default.

## Plan output

`WatchPlan` with command, target, caps, verification, cancellation.

## Execution flow

Validate target → start watcher → apply/hot-reload as configured → collect bounded output.

## Verification flow

Build/watch output, file hash/timestamp, optional console redaction.

## Rollback flow

Stop watcher; restore temporary config if any.

## Idempotency notes

The planner MUST diff current state before emitting mutations. Re-running an already satisfied desired state SHOULD become a no-op plus verification unless the user explicitly requests refresh/reapply.

## Failure modes and recovery

- Third-party package script -> block until trusted/generated.
- Watcher fails -> stop and preserve last-known-good.


## Data contracts

Primary contracts: `consent-grant, operation-run`. Any implementation-specific schema expansion MUST update `docs/content/docs/archive/add-spicetify-skill/schemas/index.mdx`, `api-contracts.md`, and regression prompts.

## Cross-cutting controls

- Policy decision is required before execution, even for no-op verification plans.
- Secrets, prefs content, logs, screenshots, and private paths follow `privacy-redaction.md`.
- Third-party, Marketplace, imported, or unknown assets require provenance and audit before enablement.
- Desired-state or automation inputs cannot waive non-negotiable safety invariants.

## Example user prompts

- `/spicetify watch my terminal theme while I edit CSS`
