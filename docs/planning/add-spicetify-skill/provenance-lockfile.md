# Provenance lockfile

**Path:** `docs/planning/add-spicetify-skill/provenance-lockfile.md`
**Purpose:** Model for pinning, auditing, accepting, and replaying third-party sources.
**Status:** Proposed
**Load/use when:** Implementing `audit`, `theme install-github-safe`, `extension`, `custom-app`, `marketplace`, or profile replay.

## Why a lockfile exists

Spicetify customizations can include executable JavaScript (`extension.js`, `theme.js`, custom apps, custom-app extension subfiles) and CSS that changes UI semantics. Marketplace and GitHub convenience must not collapse into implicit trust. A lockfile makes provenance explicit and replayable.

## Lockfile location

```text
.spicetify-skill/provenance-lock.json
```

or under the `/spicetify` state root when the implementation is not inside a repo.

## Record shape

```json
{
  "schema": "spicetify-skill.provenance-lock",
  "entries": [
    {
      "assetId": "theme:terminal-dark",
      "assetType": "theme",
      "sourceKind": "generated-local",
      "sourceUri": "local://generated/terminal-dark",
      "ref": "generated-local",
      "files": [
        { "path": "color.ini", "sha256": "..." },
        { "path": "user.css", "sha256": "..." }
      ],
      "auditId": "audit_<id>",
      "auditVerdict": "allow",
      "acceptedRisk": null,
      "installedTo": "Themes/TerminalDark",
      "installedAt": "<timestamp>"
    }
  ]
}
```

## Source kinds

| Source kind | Trust posture | Notes |
|---|---|---|
| `generated-local` | trusted after template tests | Generated by `/spicetify`; still parsed and verified. |
| `local-user` | partially trusted | User-owned file; audit before enable if executable. |
| `staged-download` | untrusted | Network fetch approved, pinned, staged, audited. |
| `marketplace-repo` | untrusted | Marketplace-compatible metadata does not imply safe execution. |
| `legacy-existing` | unknown | Existing enabled assets discovered during migration. |

## Acceptance policy

- `allow`: may install/enable after normal snapshot/confirmation.
- `warn`: requires explicit audit acceptance bound to `auditId` and file hashes.
- `block`: cannot install/enable through normal flow; may produce manual review report only.
- Changed files invalidate previous audit and acceptance.
- Accepted risk expires when source ref, hash, target path, or declared Spicetify compatibility range changes.

## Replay rules

Profile replay MUST verify:

- referenced asset exists in lockfile,
- source hashes match,
- audit verdict/acceptance is still valid,
- current policy permits the risk,
- target platform can install the asset.
